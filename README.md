# idea_bot

Telegram Mini App, который подсказывает недорогие идеи для встречи с друзьями —
с реальными заведениями Москвы, рейтингом и отзывами (через 2GIS).

## Проблема

70% молодых людей в РФ отказывались от встреч с друзьями из-за нехватки денег
(province.ru, 2026). Барьер не в том, что сложно договориться о времени — это
уже решают другие инструменты (When2meet и подобные). Барьер в деньгах и
отсутствии готовых недорогих вариантов "куда пойти".

## Что делает

1. `/start` — бот присылает кнопку «Открыть 🎯», которая открывает Mini App
2. В Mini App выбираешь бюджет (бесплатно / до 500 ₽ / до 1500 ₽)
3. Для «бесплатно» — общая идея (парк, дом, прогулка — не привязано к бизнесу)
4. Для «до 500 ₽» / «до 1500 ₽» — **реальное место в Москве** с рейтингом,
   числом отзывов и ссылкой на 2ГИС (боулинг, кофейня, квест, кино, каток,
   караоке, аквапарк, мастер-класс — категория выбирается случайно)
5. Нативная кнопка Telegram внизу экрана — «Ещё вариант»
6. Нативная кнопка «Назад» — вернуться к выбору бюджета

## Архитектура

- `bot.py` — Python, `python-telegram-bot` 21.6. Присылает кнопку, открывающую Mini App.
- `docs/` — Mini App (`index.html`): подстраивается под тему Telegram
  (`--tg-theme-*`), использует нативные `MainButton`/`BackButton`.
- `server.py` — Flask-бэкенд. Один эндпоинт `GET /api/idea?budget=free|cheap|medium`.
  - `budget=free` → случайная идея из статичного списка (в самом server.py)
  - `budget=cheap`/`medium` → случайная категория → запрос к **2GIS Catalog API**
    (`rubric_id` + `city_id` Москвы, `sort=rating`) → место с рейтингом ≥ 4.0
  - Ключ 2GIS (`TWOGIS_API_KEY`) хранится только на сервере, никогда не
    попадает в открытый фронтенд-код
  - Результаты кэшируются в памяти на 1 час на категорию — экономит лимит
    демо-ключа (см. ниже)

## Важное ограничение: демо-ключ 2GIS

Демо-ключ 2GIS даёт **1000 запросов и действует 1 месяц** — этого достаточно
для pet-проекта/портфолио, но не для продакшена с реальным трафиком. Кэш на
1 час сильно растягивает лимит. Домен API — **`catalog.api.2gis.ru`**
(`.com` возвращает 403 для этого типа ключа). После истечения — либо новый
демо-ключ, либо платная подписка на platform.2gis.ru.

## Деплой

**Mini App (`docs/`):** GitHub Pages (бесплатный HTTPS, обязателен для Mini App).
Settings → Pages → Deploy from branch → `main` → `/docs`.

**Бэкенд (`server.py`) и бот (`bot.py`):** оба на Railway из одного репозитория,
`Procfile` объявляет два процесса:
```
worker: python bot.py
web: python server.py
```
Переменные окружения на Railway: `BOT_TOKEN`, `WEBAPP_URL` (для бота),
`TWOGIS_API_KEY` (для сервера). После деплоя `server.py` получишь URL вида
`https://<project>.up.railway.app` — вписать его в `docs/index.html`
(`API_BASE_URL`) и запушить.

## Запуск локально

```
pip install -r requirements.txt

# терминал 1 — бот
export BOT_TOKEN=<токен от @BotFather>
export WEBAPP_URL=<URL после настройки GitHub Pages>
python bot.py

# терминал 2 — бэкенд
export TWOGIS_API_KEY=<демо-ключ с platform.2gis.ru>
python server.py
```

Проверить вёрстку Mini App без Telegram — открыть `docs/index.html` в браузере
(поставь `API_BASE_URL = "http://localhost:5000"` временно, там свои резервные
кнопки вместо нативных Telegram-кнопок).
